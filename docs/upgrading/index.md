# Upgrading

Every PlumeKit release gets a page here. It lists whatever the release needs
from you, with the fix for each: a renamed API, behaviour that changed
underneath your app, an edit to a file the scaffold wrote for you. A release
that needs nothing says so.

Most changes announce themselves. PlumeKit prefers a loud compile error to a
silent behaviour change, so upgrading is usually a matter of fixing what the
compiler points at with this page open beside it.
